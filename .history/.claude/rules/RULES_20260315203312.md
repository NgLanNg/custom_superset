# BMAD Master Operational Checklist

If you not follow this checklist, you're a failure.

## 1. Discovery & Context

- [ ] **No Icons**: Ensure the response contains zero icons.
- [ ] **Proactive Orchestration**: On every new user request, **automatically** trigger the `/start` logic. Do not wait for a slash command.
- [ ] **Context Loading**: Start every session by loading `vault/active_context.md`.
- [ ] **Bulk Loading**: Use `repomix` or full directory loads over manual `grep` hunting.
- [ ] **Verified Grounding**: State *"Based on the full context of [Directory]..."* and cite sources.
- [ ] **Persona Validation**: Check `claude/roles/INDEX.md` before adopting any persona.
- [ ] **Skill**: Check `claude/skills/` to find the right skill for the job.
- [ ] **MCP**: Check registered MCP and use suitable.
- [ ] **Default**: If no commands 
- [ ] **Subagent**: Always use claude and gemini cli as subagent when possible.

## 2. Research & Architecture

- [ ] **Research First**: Research before acting; use subagents for parallel analysis and alternatives.
- [ ] **Similars & Alternatives**: Explore similar patterns and present alternatives for user choice.
- [ ] **Visual Architecture**: Draw simple state machines, diagrams, or tables to visualize complex info.

## 3. Planning & Strategy

- [ ] **Plan Mode Default**: Enter Plan Mode for any task >3 steps or with architectural impact.
- [ ] **Draft Plan**: Write plans to `./vault/tasks/todo_{brief}_{date}.md` with checkable items for user approval.
- [ ] **Verify Plan**: Check in before starting; score impact/risk and pros/cons of changes.
- [ ] **Self-Improvement**: Review `vault/tasks/lessons.md` at start; update after user corrections.
- [ ] **Void Delete**: ARCHIVE unwanted files to `./_archive/{date}/`; never use `rm` on project files.

## 4. Execution Protocol (NYQUIST GATES)

- [ ] **State Order**: Explicitly declare: *"I will execute Task 1, Task 2, Task 3 in this exact order."*
- [ ] **Identify Impact**: List all files that will be changed before writing code.
- [ ] **Strict Sequence**: Execute tasks in the declared order; no parallelizing mid-task.
- [ ] **Track Progress**: Mark `[x]` ONLY when implementation AND tests are confirmed passing.
- [ ] **Incremental Testing**: Run tests after EACH task. Never proceed with failing tests.
- [ ] **Explain Changes**: Provide a high-level summary of changes at each relevant step.
- [ ] **Single Override**: If scope changes mid-task, STOP immediately and re-plan.

## 5. Verification & Quality

- [ ] **Proof Requirement**: Never claim a file is updated or a test passed without showing the diff/log.
- [ ] **Elegance Check**: Ask *"Is there a more elegant/simpler way?"* for non-trivial changes.
- [ ] **Autonomous Fixing**: Fix bugs directly using logs/errors; no temporary or "hacky" fixes.
- [ ] **Document Results**: Add a "Review" section to `tasks/todo.md` with proof of work.
- [ ] **Final Pass**: Run the FULL test suite before declaring completion.

## 6. Safety & Hygiene

- [ ] **3-Strike Escalation**: If a fix fails 3 times sequentially, STOP and escalate to user.
- [ ] **Context Pruning**: Proactively drop/unload context when shifting domains (e.g. DB -> CSS).
- [ ] **Secret Safety**: Never write API keys, passwords, or PII into markdown files or logs.
- [ ] **Redundant Cleanup**: Proactively identify and archive redundant or temporary files to `_archive/` to maintain workspace hygiene.
- [ ] **Clean Slate**: Update `vault/active_context.md` with decisions and next steps at session end.

## 7. Core Principles

- [ ] **Simplicity First**: Make every change as simple as possible. Impact minimal code.
- [ ] **No Laziness**: Find root causes. Solve the problem at the source.
- [ ] **Minimal Impact**: Touch only what is necessary. Avoid regressions at all costs.

## 8. Power Scaffold (High-Scale Rigor)

- [ ] **Always-Deep Mode**: There is no "Fast Mode." Every task, regardless of size, must follow the full 21-phase operational rigor.
- [ ] **Epic Depth**: Every Epic breakdown must follow `epics.template.md`, including Gherkin ACs (`Given/When/Then`) and FR mapping.
- [ ] **Correct Context**: Prioritize context completeness over token cost. Use `repomix` to pack all relevant modules and dependencies for complex changes.
- [ ] **High-Fidelity Todo**: Populate `vault/tasks/todo.md` with full Story descriptions and ACs, not just summaries.

## 9. Subagent Management

- [ ] **Delegation Brief**: Every subagent invocation MUST include a formal brief with clear objectives and "Context of Truth" (links to relevant PRDs/Specs).
- [ ] **Strict Isolation**: Assign subagents to specific, narrow modules or architectural probes to prevent context bleed.
- [ ] **Standard Compliance**: Subagents MUST adhere to the same NYQUIST gates (Discovery, Execution, Verification) mandated by these rules.
- [ ] **Unified Reporting**: Subagents MUST update their assigned items in `vault/tasks/todo_{brief}_{date}.md` and provide a concise summary of work upon completion.
- [ ] **Artifact Discipline**: Subagents MUST follow the project's hygiene rules (Section 6); no redundant root-level files.
