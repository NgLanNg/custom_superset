# Apache Superset Installation - Epic Breakdown (Developer Mode)

## Overview

This document outlines the steps to install Apache Superset natively from source (Developer Mode) on macOS in the `/Users/alan/dashboard` workspace. This includes installing backend Python dependencies in editable mode and building the React frontend via npm.

## Epic 1: Source Acquisition & Backend Setup

Goal: Clone the repository and configure the backend development environment.

### Story 1.1: Clone the Repo

As a Developer,
I want to grab the Superset source code,
So that I have the local context for my dev work.

**Acceptance Criteria:**

- **Given** git is installed.
- **When** I run `git clone --depth=1`.
- **Then** the `superset` folder is populated with python and JS source.

### Story 1.2: Editable Python Install

As a Developer,
I want to create a python venv and install superset with `-e .`,
So that my local python changes are instantly reflected in the server.

**Acceptance Criteria:**

- **Given** python 3.11 is present.
- **When** I run `pip install -e .[development]`.
- **Then** all python dependencies map to the source directory.

## Epic 2: Frontend Setup & Build

Goal: Download NPM dependencies and compile React assets.

### Story 2.1: NPM Install & Build

As a Developer,
I want to install node modules and build the superset frontend,
So that static assets are available or I can run webpack-dev-server.

**Acceptance Criteria:**

- **Given** node 16+ and npm 7+ are installed.
- **When** I navigate to `superset-frontend` and run `npm ci` and `npm run build`.
- **Then** front-end assets are successfully compiled into the static directory.

## Epic 3: Database Init & Development Servers

Goal: Initialize the SQLite metadata database and launch backend + frontend in dev mode.

### Story 3.1: Initialize Superset Resources

As a Developer,
I want to run `db upgrade`, `fab create-admin`, `load_examples`, and `init`,
So that Superset has all required schema and demo data.

**Acceptance Criteria:**

- **Given** Superset is installed.
- **When** I run the initialization commands via the superset CLI.
- **Then** they succeed without error and an admin is available.

### Story 3.2: Run Development Servers

As a Developer,
I want to start the Flask dev server and webpack dev server,
So that I can test the full stack with hot reloading.

**Acceptance Criteria:**

- **Given** the database is initialized.
- **When** I run `FLASK_ENV=development superset run -p 8088 --with-threads --reload --debugger`.
- **Then** the UI is reachable at `http://localhost:8088` (or 9000).
