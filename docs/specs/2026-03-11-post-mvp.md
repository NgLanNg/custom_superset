# Post-MVP Features Spec
Date: 2026-03-11

## 1. dbt Gold model (`gold_scenario_equity_share.sql`)
**Objective**: Create a deduplicated target view/table for reporting.
- **Given** `silver_scenario_equity_share` contains multiple versions of a scenario (appended writes or overlapping updates).
- **When** the dbt model runs.
- **Then** it should use `ROW_NUMBER() OVER (PARTITION BY scenario_name, bu, opu, year ORDER BY updated_at DESC)` to yield only the most recent row per coordinate.

## 2. Tableau connection to Gold model
**Objective**: Instruct/document how Tableau connects to the gold model.
- **Note**: Will be documented in `docs/system/project-overview.md` if external configuration is needed.

## 3. OPU Configuration Tab
**Objective**: Move from placeholder to a real OPU configuration view.
- **Given** the user navigates to the "OPU Configuration" tab in the Scenario Chart.
- **When** the tab is rendered.
- **Then** is should display the real interactive configuration or proper data instead of a placeholder message.

## 4. Growth Configuration Tab
**Objective**: Replace mock random pipeline data with real data or an empty state for real data.
- **Given** the "Growth Configuration" tab.
- **When** the Pipeline Manager table is shown.
- **Then** it must not use `Math.random()`. It must connect to `growthData` properties or display actual tracked properties (Capacity, Capex, Status).

## 5. Production RDS connection config
**Objective**: Add configuration for Production RDS.
- **Given** deployment to production.
- **When** connecting to the database.
- **Then** Superset should be able to connect to `peth_prod`.
