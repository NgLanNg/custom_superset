# Step 01: Define Scope

## 1. Goal

Read the context from `docs/project-context.md` or from the user's prompt. Summarize the overarching goal of the implementation in 2-3 sentences. Identify exactly what problem is being solved.

## 2. Boundaries

Clearly define:

- **In-Scope:** The specific features, screens, models, or tasks that need to be accomplished.
- **Out-of-Scope:** Explicitly state what is NOT being built to prevent feature creep.

## 3. Success Criteria

Define 3-5 bullet points that, if true, mean the developer's work is finished and successful.

## 4. Draft the Document

Generate the content and ask the user for approval. Once approved, write it entirely to `docs/scope.md`.
