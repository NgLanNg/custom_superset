# /start

**Purpose: Automatically triggered or manually called to bootstrap the agent session. It ensures context is loaded and routes immediately to the correct workflow.**

**Scope:** `daily`
**Phase:** Multi-phase
**Deep Mode:** (Routing command — delegates to `/help`)

## When to Use

- At the beginning of every session (automatically triggered by BMAD rules).
- When context feels stale and you need a "grounding" scan.

## Execution Path
>
> **Mandatory Session Bootstrapping**

1. **Context Loading**:
   - Load `vault/active_context.md` to see the current mission.
   - Load `vault/tasks/todo.md` to check open items.
2. **Project Synchronization**:
   - Verify environment variables (e.g., `SUPERSET_SECRET_KEY`).
   - Run a quick health check if applicable.
3. **Master Routing**:
   - Delegate immediately to `/help` for phase-specific discovery.

## Output

- Fully grounded agent ready for rigorous execution.
- Redirection to the next action (e.g., "Resuming `/sprint`").

## Key Principles

- **Automatic by Default**: In BMAD, we don't wait for permission to start properly.
- **Context First**: Never act before reading the `vault`.
- **Rigor Check**: If the `todo.md` is empty, `/start` should push towards `/research` or `/scan`.
