# Custom Superset Editable Grid - Epic Breakdown

## Overview

This document outlines the epics and stories required to build a custom "inputable" table using the AG Grid plugin out of Apache Superset. The main goals are to enable embedding (skip the left navbar) and write data changes back to the backend.

## Epic 1: Enable Standalone Embedding & Config overrides

Goal: Configure Superset to function as an embeddable component.

### Story 1.1: Global Configuration

As a Developer,
I want to configure CORS, Feature Flags, and standalone parameters,
So that external applications can embed the Superset iframe without the navbar.

**Acceptance Criteria:**

- **Given** I open an iframe to Superset.
- **When** the URL query string contains `?standalone=1` or `?standalone=2`.
- **Then** the left navbar and top header are removed.
- **And** Cross-Origin access is granted via `superset_config.py`.

## Epic 2: Enhance AG-Grid Plugin for Input

Goal: Make the AG Grid cells editable so users can interact with the table.

### Story 2.1: Frontend React Implementation

As a Developer,
I want to add `editable: true` and an `onCellValueChanged` event to the AG-Grid chart plugin,
So that cells become inputs.

**Acceptance Criteria:**

- **Given** a user loads the AG-Grid chart.
- **When** they double-click a cell.
- **Then** the cell turns into a text/number input.
- **And** pressing Enter fires an API call to save the changes.

## Epic 3: Write-back Backend Endpoint

Goal: Provide an API to ingest modifications from the AG Grid frontend.

### Story 3.1: Python REST Endpoint for Write-backs

As a Developer,
I want to create a `/api/v1/writeback` (or similar endpoint),
So that the input values from the AG Grid table are actually saved.

**Acceptance Criteria:**

- **Given** the frontend sends a POST request with row data mapping.
- **When** the backend receives the payload.
- **Then** it authenticates the user and executes a SQL `UPDATE` or internal logic to persist the change.
