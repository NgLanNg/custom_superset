# Active Context

## Current Objective

Build the **Scenario Creation** feature as a native `@superset-ui` chart plugin inside Apache Superset.

The plugin renders an editable equity-share grid (existing assets + growth projects) with a live ECharts line chart. Cell edits are written back to **PostgreSQL** (`silver_scenario_equity_share`) via a custom Flask Blueprint (`ScenarioWritebackView`).

## Key Decisions

- **Architecture:** Option C -- Native Superset Plugin (not standalone iframe / not Superset-fork)
- **Dev Database:** PostgreSQL 16 (local Docker via `docker-compose.yml`) -- `superset` database
- **Production Database (planned):** PostgreSQL RDS at `peth_prod.silver_scenario_equity_share`
- **PK Type:** UUID (`gen_random_uuid()` default in PostgreSQL)
- **Write-back Auth:** `@has_access_api` decorator + `SupersetClient.post()` -- no CSRF exemptions
- **Optimistic UI:** React updates local state immediately on cell edit; persists to DB in background; reverts on failure
- **Blueprint Registration:** `FLASK_APP_MUTATOR` in `superset_config.py` (deferred import pattern)
- **Config Override:** `SUPERSET__SQLALCHEMY_DATABASE_URI` env var required for CLI commands (`superset db upgrade`) to correctly target PostgreSQL
- **`file_path = NULL`:** Distinguishes manually-entered UI rows from Lambda-ingested rows (audit trail)
