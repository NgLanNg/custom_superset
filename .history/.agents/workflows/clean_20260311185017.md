# /clean

**Purpose: Maintain a high-signal workspace by proactively archiving redundant logs, temporary scripts, and outdated context files.**

**Scope:** `daily`
**Phase:** Phase 7 — Maintenance
**Deep Mode:** (Hygiene maintenance)

## When to Use

- At the end of a session or sprint.
- When the root directory feels cluttered with `.tmp`, `.log`, or `_old` files.
- Before a `/scan` to ensure the LLM context isn't diluted by noise.

## Execution Path
>
> **Mandatory Hygiene Protocol**

1. **Identification**: Locate all files in the root or `/tmp` that are not part of the core project structure (e.g., `temp_debug.py`, `output.txt`, previous version backups).
2. **Archival**:
   - Move identified files to `./_archive/{YYYY-MM-DD}/`.
   - Never use `rm` on implementation files; always archive first.
3. **Log Rotation**: Consolidate or archive large log files from `./logs/` or `.agents/logs/`.
4. **Context Pruning**: Review `vault/active_context.md` and move completed milestones to `vault/reports/`.

## Output

- A clean, high-signal workspace.
- Archived artifacts safely stored in `./_archive/`.

## Key Principles

- **No Data Loss**: Archive everything before deleting.
- **Signal Over Noise**: A clean repo leads to better LLM reasoning.
- **Predictable Structure**: Always use date-based folders in `./_archive/`.
