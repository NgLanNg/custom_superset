# Workflow: Brainstorm (Phase 1)

## 🏗️ WORKFLOW ARCHITECTURE

This phase uses **micro-file architecture** to ensure a disciplined discovery process. The brainstorming workflow is divided into three focused steps:

1. **Connect**: Understand the core problem and constraints (`step-01-connect.md`).
2. **Ideate**: Explore approaches using structured frameworks (`step-02-ideate.md`).
3. **Synthesize**: Score approaches and form a concrete design (`step-03-synthesize.md`).

- Each step is self-contained in the `steps/` directory.
- Progressive refinement through a "Step-File" pattern.
- Collaborative dialogue focusing on "One question at a time".

## 🚀 Execution

To begin this workflow, load and follow:
`{.agents}/phases/phase-1-analysis/00-brainstorm/steps/step-01-connect.md`

## 🏁 Phase End State

- `docs/plans/YYYY-MM-DD-{topic}-design.md` is generated.
- Initial consensus reached on architectural direction or feature set.
- Ready for PRD creation or Architecture design.
