# Active Context

## Current Objective

Build the **Scenario Creation** feature as a native `@superset-ui` chart plugin inside Apache Superset.

The plugin renders an editable equity-share grid (existing assets + growth projects) with a live chart. Cell edits are written back to **PostgreSQL RDS** (`peth_prod.silver_scenario_equity_share`) via a custom Flask Blueprint, following existing Silver/Gold dbt conventions. A dbt gold model (`gold_scenario_equity_share`) makes the data available to Tableau.

## Key Decisions

- **Architecture:** Option C — Native Superset Plugin (not standalone iframe / not Superset-fork)
- **Database Target:** `peth_prod.silver_scenario_equity_share` on PostgreSQL RDS — NOT the Superset metadata SQLite DB
- **PK Type:** `uuid` with `gen_random_uuid()` — matches all silver tables
- **Write-back Auth:** Uses Superset's native `@protect()` decorator + `SupersetClient.post()` on the frontend — no CSRF exemptions or custom tokens needed
- **Optimistic UI:** React updates local state immediately on cell edit; persists to DB in background; reverts on failure
- **dbt Gold layer:** `gold_scenario_equity_share.sql` deduplicates by `ROW_NUMBER() OVER (...ORDER BY updated_at DESC)` — same pattern as all existing gold models
- **`file_path = NULL`:** Distinguishes manually-entered UI rows from Lambda-ingested rows (audit trail)
