# Apache Superset Installation - Epic Breakdown

## Overview

This document outlines the steps to install Apache Superset natively from scratch on macOS (using a Python 3.11 virtual environment) in the `/Users/alan/dashboard` workspace.

## Epic 1: OS Dependencies & Environment Setup

Goal: Ensure the host has the required software to build Superset's Python dependencies.

### Story 1.1: Install Homebrew Packages

As a Developer,
I want to install Homebrew packages (python@3.11, openssl, pkg-config),
So that I have the required system boundaries to build binary wheels.

**Acceptance Criteria:**

- **Given** Homebrew is installed.
- **When** I run `brew install ...`.
- **Then** Python 3.11 and necessary C-extensions libraries are present.

### Story 1.2: Create Python Virtual Environment

As a Developer,
I want to create a Python 3.11 virtual environment,
So that I don't pollute the global python site-packages.

**Acceptance Criteria:**

- **Given** Python 3.11 is installed.
- **When** I run `python3.11 -m venv venv`.
- **Then** a `.venv` or similar folder is created.

## Epic 2: Installation & Configuration

Goal: Download and configure Superset via pip.

### Story 2.1: Pip Install Apache-Superset

As a Developer,
I want to install `apache-superset` using pip,
So that all Python dependencies are resolved.

**Acceptance Criteria:**

- **Given** the virtual environment is active.
- **When** I run `pip install apache-superset`.
- **Then** no build errors should occur.

### Story 2.2: Configure FLASK_APP and SECRET_KEY

As a Developer,
I want to export `FLASK_APP` and set a `SUPERSET_SECRET_KEY`,
So that the Flask commands and UI run without errors.

**Acceptance Criteria:**

- **Given** pip has installed Superset.
- **When** I set `export FLASK_APP=superset` and `export SUPERSET_SECRET_KEY='YOUR_SECRET_KEY'`.
- **Then** commands like `superset` run successfully.

## Epic 3: Database Init & Verification

Goal: Initialize the SQLite metadata database and confirm access.

### Story 3.1: Initialize Superset Resources

As a Developer,
I want to run `db upgrade`, `fab create-admin`, `load_examples`, and `init`,
So that Superset has all required schema and demo data.

**Acceptance Criteria:**

- **Given** Superset is installed.
- **When** I run the initialization commands.
- **Then** they succeed without error and an admin is available.

### Story 3.2: Run and Verify UI

As a Developer,
I want to start the Superset development server,
So that I can verify it runs on port 8088.

**Acceptance Criteria:**

- **Given** the database is initialized.
- **When** I run the dev server.
- **Then** the UI is reachable at `http://localhost:8088`.
