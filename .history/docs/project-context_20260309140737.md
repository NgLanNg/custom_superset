# Project Context: Custom Superset

## 1. Technology Stack

- **Backend**: Python 3.11, Flask, SQLAlchemy.
- **Frontend**: Node.js (v24), React (TypeScript), Webpack, `@superset-ui` core architecture.
- **Database**: SQLite (currently for metadata/examples, typically Postgres in prod).
- **Key External Library**: `@deck.gl`, `ag-grid-react` (Table plugin), Echarts.

## 2. Implementation Patterns

- **Monorepo Structure**: Separation into `superset/` (backend) and `superset-frontend/` (frontend node ecosystem).
- **Feature Flags**: Application behaviors are controlled strictly via backend configurations and JS feature flags mapped at runtime.
- **Plugin Architecture**: Visualizations are registered as distinct `@superset-ui/plugin-*` packages internally.

## 3. Strict Implementation Rules

> [!CRITICAL]
> These rules must be adhered to by all automated agents working on this project.

1. **Embedding**: To skip navbars and embed securely, rely on the official `EMBEDDED_SUPERSET` configuration alongside [Guest Tokens] and the `@superset-ui/embedded-sdk` rather than hacking CSS to hide navbars.
2. **Editable Grids**: The `plugin-chart-ag-grid-table` plugin must be fully leveraged instead of legacy tables to provide spreadsheet-like interfaces or editable behaviors.
3. **Configuration**: ALL environment variables and Superset configuration extensions should reside in `superset_config.py` rather than mutating the core `config.py` file to ensure upgrade paths.
