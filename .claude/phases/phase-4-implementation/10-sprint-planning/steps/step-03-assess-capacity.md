# Step 03: Assess Capacity

**Purpose:** Calculate sprint capacity and allocate appropriate stories.

## Instructions

1. Review team availability for the sprint period
2. Estimate each ready story with T-shirt sizes (S, M, L)
3. Select stories that fit within capacity bounds
4. Respect dependency ordering (backend before frontend)

## Output

- `capacity_estimate`: Total points/hours available for sprint
- `story_estimates`: T-shirt estimates for all selected stories
- `allocated_stories`: Final selection for the sprint
