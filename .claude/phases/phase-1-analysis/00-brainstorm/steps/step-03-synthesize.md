# Step 03: Synthesize & Design

## STEP GOAL

Consolidate the ideas and constraints explored during ideation into a concrete design or architectural direction.

## MANDATORY EXECUTION RULES (READ FIRST)

- 🛑 NEVER generate content without user input.
- 📖 CRITICAL: Read the complete step file before taking any action.
- 📋 YOU ARE A FACILITATOR, not a content generator.
- ✅ YOU MUST ALWAYS SPEAK OUTPUT In your Agent communication style.

## EXECUTION PROTOCOLS

- 🎯 One question at a time.
- 🎯 YAGNI ruthlessly.
- 💾 Save the final design to `docs/plans/YYYY-MM-DD-{topic}-design.md`.

## CONTEXT BOUNDARIES

- Available context: Current workspace, `./vault/active_context.md`, and findings from Step 01 and Step 02.
- Focus: Solution design, architectural choices, and feature scoring.
- Limits: The output is an initial design doc, not a comprehensive PRD.

## 📋 Steps

### A) Scoring and Synthesis

1. If applicable from Step 02, score generated approaches using: (Impact × Feasibility) / Risk.
2. Synthesize the findings into a clear, single recommendation or direction.
3. Present the recommendation to the user.
4. Wait for feedback before proceeding.

### B) Presenting the Design

1. Draft the initial design based on the synthesis.
2. Present the design in 200-300 word chunks.
3. Validate each section with the user.
4. Save the final finalized design to `docs/plans/YYYY-MM-DD-{topic}-design.md`.
