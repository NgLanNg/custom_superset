# Step 01: Break Down Tasks

## 1. Context Read

Read `docs/scope.md` to internalize what will be built. If `docs/architecture.md` exists, cross-reference it for technical dependencies.

## 2. List Generation

Define work sequentially. Write directly to `docs/tasks.md`.

Format per task:

```markdown
## Task: [Name]
- Status: ready | in-progress | done
- Depends on: [other task or "none"]
- Acceptance: [one-line definition of done]
```

## 3. Review

Present the generated list to the user to ensure it comprehensively covers the Scope defined in Step 01.

## ⏭️ Advance/Party/Continue (A/P/C)

- [A] - Advance to `Phase 3: Solutioning`
- [P] - Party! Update context and lessons learned
- [C] - Continue refinement of tasks
