# Apache Superset Installation - Epic Breakdown

## Overview

This document outlines the steps to "pull and install" Apache Superset in the `/Users/alan/dashboard` workspace using Podman/Podman-compose.

## Epic 1: Repository Acquisition

Goal: Clone the official Apache Superset repository into the dashboard workspace.

### Story 1.1: Clone Superset Repository

As a Developer,
I want to clone the Apache Superset source code,
So that I have the necessary Docker/Podman configuration files locally.

**Acceptance Criteria:**

- **Given** Git is installed.
- **When** I run `git clone --depth=1 https://github.com/apache/superset.git`.
- **Then** a `superset` directory should exist in the root.
- **And** the `.git` history should be minimal (shallow clone).

## Epic 2: Installation & Deployment

Goal: Deploy Superset using podman-compose.

### Story 2.1: Configure Environment

As a Developer,
I want to configure the environment variables for Superset,
So that it runs correctly with a secure secret key.

**Acceptance Criteria:**

- **Given** the repository is cloned.
- **When** I create/update `.env` files if needed.
- **Then** a `SUPERSET_SECRET_KEY` should be set.

### Story 2.2: Launch Superset Services

As a Developer,
I want to start the Superset services using podman-compose,
So that the application becomes accessible.

**Acceptance Criteria:**

- **Given** Podman and Podman-compose are installed.
- **When** I run `podman-compose up -d`.
- **Then** all required containers (db, redis, superset) should be running.

## Epic 3: Verification

Goal: Ensure the installation is successful and functional.

### Story 3.1: Database Initialization & Admin User

As a Developer,
I want to initialize the database and create an admin user,
So that I can log into the application.

**Acceptance Criteria:**

- **Given** the services are running.
- **When** I run the `superset-init` script or equivalent exec commands.
- **Then** the database should be migrated, and the 'admin' user should be created.

### Story 3.2: Health Check

As a Developer,
I want to verify the web UI is accessible,
So that I know the installation is complete.

**Acceptance Criteria:**

- **Given** the initialization is done.
- **When** I access `http://localhost:8088`.
- **Then** the login page should load.
