# Active Context

## Current Objective

Customize Apache Superset to function as an embeddable, standalone component (skipping the left navbar) and enhance the Ag-Grid table plugin to support "inputable"/editable cells, matching the provided UX reference.

## Current Phase

Phase 2: Planning & Architecture

## Key Decisions

- Created `docs/project-context.md` via the `/init` workflow bootstrap.
- Created `superset_config.py` to house our custom configuration (`EMBEDDED_SUPERSET=True`, `AG_GRID_TABLE_ENABLED=True`, and cross-origin embedding headers).
- **Embedding / No Navbar**: Superset supports a "standalone" mode parameter (`?standalone=true` or `?standalone=1` / `2` ) in its URLs which naturally trims the navbar, allowing seamless integration into host React apps using `@superset-ui/embedded-sdk`.
- **Inputable Table**: Standard Superset AG-Grid is read-only because Superset is designed for OLAP analytics. To make cells "inputable" and persist data back, we must either:
  1. Add a custom generic backend endpoint in Flask to receive the cell edit events.
  2. Modify the frontend `@superset-ui/plugin-chart-ag-grid-table` to make the grid cells editable (`editable: true` in the columnDefs) and attach an `onCellValueChanged` event.
