# Phase 4: Course Correction

## Execution Checklist

### Step 01: RCA & Diagnosis

- [ ] Run `step-01-diagnose.md` to identify the root cause of the shift or failure.
- [ ] Determine if the issue is Technical, Requirement-based, or Infrastructure-related.
- [ ] Detail the exact delta between the original plan and the current reality.

### Step 02: Re-Solution & Proposal

- [ ] Draft the `docs/change_proposal.md` with new success criteria.
- [ ] Identify which stories need to be scoped down or timeline-extended.
- [ ] Receive user approval on the updated path forward.

### Step 03: Re-Planning

- [ ] Update the `sprint_plan.md` to match the approved proposal.
- [ ] Re-Solution any epics impacted by the change.

- [ ] **NYQUIST GATE**: Verify the new plan is technically sound and avoids the pitfalls that triggered the correction.
