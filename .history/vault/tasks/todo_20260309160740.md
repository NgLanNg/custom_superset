## Epic 2: Backend Flask API

### T3 — Rewrite `scenario_writeback.py` Blueprint

- **File:** `superset/superset/views/scenario_writeback.py`
- **Action:** Replace the current stub (which targets wrong fields and has no real DB write) with the full implementation from design doc §4:
  - Import `superset.db`, `flask_login.current_user`, `sqlalchemy.dialects.postgresql.insert`
  - Validate all required fields: `scenario_name`, `bu`, `opu`, `year`, `value`
  - Validate `value` is in range 0–100
  - Perform a PostgreSQL `ON CONFLICT DO UPDATE` UPSERT
  - Return `{"status": "success", "data": {...}}` on 200
- **Done when:** `curl -X POST http://localhost:8088/api/v1/scenario/writeback -d '{"scenario_name":"existing_assets","bu":"LNGA","opu":"MLNG","year":2020,"value":90}' -b <cookie>` returns 200 and the row exists in `peth_dev`.
- **Risk:** Medium — SQLAlchemy UPSERT dialect is PostgreSQL-specific
- **Status:** [ ]

### T4 — Confirm Blueprint Registration in superset_config.py

- **File:** `superset_config.py`
- **Action:** The `EXTRA_BLUEPRINTS` pattern is already stubbed in `superset_config.py`. Verify `scenario_bp` is correctly imported after T3 rewrite and Superset does not throw on startup.
