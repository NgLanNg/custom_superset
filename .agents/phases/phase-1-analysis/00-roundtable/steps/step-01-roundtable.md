# Step 01: Roundtable Discussion

## STEP GOAL

Facilitate a multi-persona roundtable discussion to analyze the problem from different expert perspectives.

## MANDATORY EXECUTION RULES (READ FIRST)

- 🛑 NEVER generate content without user input.
- 📖 CRITICAL: Read the complete step file before taking any action.
- 📋 YOU ARE A FACILITATOR, not a content generator.
- ✅ YOU MUST ALWAYS SPEAK OUTPUT In your Agent communication style.

## EXECUTION PROTOCOLS

- 🎯 Ensure each persona speaks strictly from their domain expertise.
- 🎯 Synthesize the discussion when all personas have spoken.
- 💾 Output the final synthesis to the user.

## CONTEXT BOUNDARIES

- Available context: Current workspace and `./vault/active_context.md`.
- Focus: Multi-perspective analysis.
- Limits: Do not write implementation code; focus on architectural and strategic decisions.

## 📋 Steps

### A) Identify Personas

1. Based on the problem statement, identify 2-4 expert personas (e.g., Security Expert, Frontend Architect, Database DBA).
2. Confirm the selected personas with the user.

### B) Conduct the Roundtable

1. Ask the user for the first topic or prompt.
2. Have each persona provide their insight, one at a time.
3. Allow personas to critique each other's points constructively.

### C) Synthesis

1. Summarize the key takeaways and consensus reach by the roundtable.
2. Present the synthesis to the user for final approval.

- [A] - Advance to next phase

- [C] - Continue the roundtable with a new topic
