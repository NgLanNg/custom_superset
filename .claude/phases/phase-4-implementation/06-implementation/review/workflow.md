# Workflow: Code Review

## 🏗️ WORKFLOW ARCHITECTURE

This workflow provides a systematic pre-commit quality and security audit.

1. **Ingest**: Identify intent, scope, and risk.
2. **Review**: Use the severity-based checklist (Critical/High/Medium/Low).
3. **Verify**: Run project-specific lints and tests.
4. **Report**: Generate a concise feedback report.

## 🚀 Execution

Load and follow the steps in:
`{.agents}/phases/phase-4-implementation/06-implementation/review/step-01-ingest.md`

## 🏁 Phase End State

- Code review report generated.
- Critical/High issues addressed or discussed.
- Ready for commit or merge.
